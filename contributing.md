# Contribution à la documentation de Adobe Experience Manager

Merci d’avoir choisi de contribuer à la documentation de Adobe Experience Manager ! Nous acceptons les contributions de contenu de l’ensemble de notre communauté de lecteurs ainsi que des autres employés d’Adobe.

Ce fichier décrit comment apporter votre contribution et indique les directives à suivre lors de l&#39;exécution de vos contributions.

## Code de conduite d’Adobe Open Source

Ce projet a adopté le [code de conduite d’Adobe Open Source](code-of-conduct.md). En contribuant à la documentation d’AEM, vous acceptez de respecter ce code de conduite. Veuillez signaler tout comportement inacceptable à [&#128279;](mailto:Grp-opensourceoffice@adobe.com).

## Instructions relatives à la contribution

Bien que nous soyons ouverts à toute contribution visant à améliorer la documentation d’AEM, nous demandons que les contributions respectent certaines directives et normes dans la mesure du possible. Pour plus d’informations, consultez les [consignes de contribution à la documentation d’AEM](guidelines.md).

## Comment contribuer

Nous voulons faciliter autant que possible la soumission de vos contributions. Vous pouvez contribuer de deux façons :

* [Créer un événement](#create-an-issue)
* [Création d’une demande d’extraction](#create-a-pull-request)

La méthode choisie dépend de votre niveau de confort dans la modification de la documentation dans GitHub.

>[!NOTE]
>
>Les contributions que vous envoyez sont couvertes par les [conditions d’utilisation d’Adobe.](https://www.adobe.com/fr/legal/terms.html)

### Créer un événement {#create-an-issue}

Avez-vous une suggestion, une observation au sujet des documents actuels, ou une question sur la façon dont quelque chose est décrit ? Créez ensuite un événement. Les problèmes sont également excellents si vous avez quelque chose à apporter, mais que vous n’êtes pas à l’aise pour modifier les documents ou effectuer vous-même une demande d’extraction.

Les problèmes vous permettent de décrire facilement vos idées d’amélioration des documents, avant examen par un membre de l’équipe de documents d’AEM. Il vous suffit de cliquer sur le lien **Enregistrer un problème** en haut du panneau de navigation de droite sur n’importe quelle page pour créer un problème.

### Création d’une demande d’extraction {#create-a-pull-request}

Si vous rencontrez un problème et savez exactement comment le résoudre, pensez à créer une requête d’extraction. Les demandes d’extraction permettent d’apporter vos propres modifications à la documentation, avant examen par un membre de l’équipe de documents AEM en vue de leur inclusion. Il vous suffit de cliquer sur le lien **Modifier cette page** en haut du panneau de navigation de droite de n’importe quelle page pour apporter les modifications suggérées et créer la demande d’extraction.

Toutes les demandes d’extraction doivent être accompagnées d’un contrat de licence du contributeur ou d’un contrat de licence du contributeur [CLA](https://opensource.adobe.com/cla.html) signé.  Cela donne à Adobe l’autorisation de redistribuer vos contributions dans le cadre du projet. Vous n’avez à envoyer un contrat de licence du contributeur Adobe qu’une seule fois. Par conséquent, si vous en avez déjà envoyé un, tout est déjà en ordre.

### Contributions des employés d’Adobe

Si vous êtes un employé d’Adobe et que vous avez une contribution, veuillez utiliser l’organisation `AdobeDocs` dans le référentiel Git de l’entreprise.

Bien entendu, nous acceptons volontiers toute contribution et examinerons votre contribution si elle est effectuée sur le référentiel public. Toutefois, les demandes d’extraction provenant des employés d’Adobe sont plus faciles à traiter et peuvent être approuvées plus rapidement sur le référentiel d’entreprise. Veillez à ce que toute information exclusive ne soit abordée que dans le référentiel d&#39;entreprise.
